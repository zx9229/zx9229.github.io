在Windows下应当安装Git到哪个目录

Git安装包(比如Git-2.18.0-64-bit.exe)会默认安装到C:\Program Files\Git或C:\Program Files (x86)\Git下。
此时路径包含了空格等特殊字符。如果用Git配合MinGW编译程序时，可能会遇到问题。
因此建议安装Git到C:\Program_Files_x64\Git或C:\Program_Files_x86\Git下。

GIT_DISCOVERY_ACROSS_FILESYSTEM not set

原因：你git init的目录(.git文件夹所在的目录)和”(要add的那个文件)所在的目录”不在同一个挂载点上。
解决：设置一个环境变量 GIT_DISCOVERY_ACROSS_FILESYSTEM=1 即可。
